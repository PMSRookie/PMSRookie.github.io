    mybatis-plus 介绍

    mybatis-plus 入门使用

    1. 新建 spring boot 应用
    2. 导入 spring boot 整合 mybatis-plus pom 依赖
    3. 导入 lombok pom 依赖 (需要安装 lombok 插件)
    • 安装 lombok 插件 : File -> Setting -> Plugins -> 搜索 lombok -> 点击 install
    1. spring boot 配置文件 (application.yml) 配置 mysql 连接相关配置
    • 可配置 mybatis-plus log (日志)
    1. 创建 table 映射对象 对象名必须和 table name 中的英文一致，符号可省略。例如：table name : user_info 则 bean name : UserInfo
    2. 创建 mapper 接口并且继承 BaseMapper<BeanName> 里面不需要写任何抽象方法
    • 例如 (记住加 @mapper 注解，或者在 spring boot 主类加 @MapperScan() 注解。)
    1. 最后去到 test 类测试，代码如下 (selectList(null) 方法参数为 null 时查询全部)

    mybatis-plus 基础操作

    1. 查询操作
      1. T selectById(Serializable id); 按照id查询单个
      2. T selectOne(@Param("ew") Wrapper<T> queryWrapper); 按照条件查询单个
      3. List<T> selectList(@Param("ew") Wrapper<T> queryWrapper); 按照条件查询多个，当参数为null时为查询全部
      4. Integer selectCount(@Param("ew") Wrapper<T> queryWrapper); 按照条件查询，返回得到的记录数
      5. List<T> selectByMap(@Param("cm") Map<String, Object> columnMap); 按照条件查询多个
      6. List<T> selectBatchIds(@Param("coll") Collection<? extends Serializable> idList); 按照多个id查询多个(批量查询)。
      7. IPage<T> selectPage(IPage<T> page, @Param("ew") Wrapper<T> queryWrapper); 分页查询
    2. 更新(修改)操作
      1. int updateById(@Param("et") T entity); 按照id更新(修改)，返回被修改修改的记录数
      2. int update(@Param("et") T entity, @Param("ew") Wrapper<T> updateWrapper); 按条件进行更新，返回被修改修改的记录数
    3. 删除操作
      1. int deleteById(Serializable id); 按照id删除，返回得到的记录数
      2. int deleteBatchIds(@Param("coll") Collection<? extends Serializable> idList); 按照多个id删除(批量删除)，返回被修改修改的记录数
      3. int deleteByMap(@Param("cm") Map<String, Object> columnMap); 按照条件进行删除，返回被修改修改的记录数
      4. int delete(@Param("ew") Wrapper<T> wrapper); 按照条件进行删除，返回被修改修改的记录数
    4. 插入操作
      1. int insert(T entity);
    • Wrapper<T> 可根据自定义 sql 语句操作
    • Map<String, Object> 只能 put(字段名, 值) 操作

    mybatis-plus 自动装配

    介绍

    mybatis-plus 自动装配是在进行操作插入或者更新时，对于一些字段 ( 它们是可以通过一些相同代码实现赋值或者赋相同值，例如：createTime，updateTime等) ，这些字段就可以使用自动装配来防止多余相同赋值代码。

    实现

    1. 给这些字段添加注解 @TableField() 并且设置参数 fill
      1. fill 参数
        1. FieldFill.DEFAULT
        2. FieldFill.INSERT : 插入操作时装配
        3. FieldFill.UPDATE : 更新操作时装配
        4. FieldFill.INSERT_UPDATE : 插入更新操作都装配
    2. 创建一个类 ( 不要忘了将类注入(@Component)到容器当中，否则spring boot不会加载这个类 )，并且实现 MetaObjectHandler 接口，并且实现接口里面的 insertFill 方法和 updateFill 方法。
    3. 在 insertFill 方法和 updateFill 方法里加入装配逻辑即可。
    4. 最后去 test 类测试即可

    mybatis-plus 乐观锁

    介绍

    1. mybatis-plus 乐观锁通过字段 version 实现。
    2. 通过控制 version 的值

    实现

    1. 在 table 及其对应映射类加上 version 字段，并且在变量上加上 @Version 注解。
    2. 在 headler 类中加入自动装配 @TableField(fill = FieldFill.INSERT)
    3. 在 mybatis-plus 配置类( @Configuration )中加上 乐观锁插件，并且注入( @Bean )到容器当中。
    4. 最后在 test 类中测试即可

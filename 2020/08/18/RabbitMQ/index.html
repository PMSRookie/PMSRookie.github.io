1. 安装

docker 安装并且运行

docker run -it --rm --name rabbitmq -e RABBITMQ_DEFAULT_USER=admin -e RABBITMQ_DEFAULT_PASS=admin -p 5672:5672 -p 15672:15672 rabbitmq:3-management

docker 运行

docker run -d --name rabbitmq -e RABBITMQ_DEFAULT_USER=admin -e RABBITMQ_DEFAULT_PASS=admin -p 5672:5672 -p 15672:15672 rabbitmq:3-management

RABBITMQ_DEFAULT_USER 设置账户

RABBITMQ_DEFAULT_PASS 设置密码

其他安装

官网网址：https://www.rabbitmq.com/download.html

注意：先安装 Erlang 语言

2. 工作模式

  1. Work queues (工作队列模式)
    • 多个消费者共同接受一条消息队列，一个消息只能由一个消费者接收。转发机制为轮询机制 (c1 -> c2 -> c3 -> c1)
  2. Publish/subscribe (发布订阅模式)
    • 一个交换机转发消息给多个消息队列，每个队列接收的消息都一样
  3. Routing (路由模式)
    • 通过匹配 routingKey 比配转发，routingKey 完全相同则可转发
  4. Topics (通配符模式)
    • 与 Routing 差不多，区别在于 routingKey 的匹配。Topics 的 routingKey 比 Routing 多了通配符。
    • 通配符 * : 可匹配单个，例如 inform.* 可匹配 inform、inform.sms、inform.email
    • 通配符 # : 可匹配单个或者多个，例如 inform.# 可匹配 inform、inform.sms、inform.email、inform.sms.email
  5. headers
    • 取消了 routingKey ，通过 key/value 键值对进行匹配
  6. RPC
    • 客户端远程调用服务端的方法
